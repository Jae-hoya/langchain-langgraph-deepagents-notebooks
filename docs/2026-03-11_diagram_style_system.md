# Diagram Style System — 2026-03-11

이 문서는 `book/assets/diagrams/*.mmd`에 공통으로 적용할 **다이어그램 시각 규칙**을 정리한 내부 스타일 가이드다.

목표:
- 장이 달라져도 독자가 색만 보고 노드의 역할을 빠르게 추론할 수 있게 한다.
- Mermaid 다이어그램의 의미 체계를 책 전체에서 일관되게 유지한다.

---

## 1. 색상 의미 규칙

### 1) 민트 (`#83CED2`)
**의미:** 기본 엔티티, 정상 단계, 주요 개념, 시작점, 결과 노드

적용 예:
- LangChain / 기본 개념 노드
- 입력 노드
- 정상 실행 단계
- 최종 응답 / 결과

### 2) 코랄 (`#ED7969`)
**의미:** 에이전트의 핵심 실행, active processing, 강한 주의가 필요한 핵심 동작

적용 예:
- Agent / Model / Generate / Execute
- 활성 처리 단계
- 실제 작업을 수행하는 핵심 노드

### 3) 노랑 (`#FFD166`)
**의미:** 판단, 제어, 조정, 승인, 중간 decision/control point

적용 예:
- Router
- Check query
- Review / HITL
- Middleware controller
- Backend / coordination layer

### 4) 회색/흰색 (`#F4F4F4`, `#FFFFFF`)
**의미:** 보조 정보, 메타데이터, 설명 노드, 외부 도구, note, 보조 경로

적용 예:
- 설명성 노드
- 외부 시스템 / support element
- note / annotation
- 상세 구분 라벨

---

## 2. 노드 의미 규칙

### 시작 / 종료
- 가능하면 민트
- `font-weight:bold` 허용

### 실행 노드
- 기본적으로 코랄
- 단, “정상 결과 생성 후 산출물 전달”은 민트로 둘 수 있음

### 분기 / 판단 노드
- 노랑 또는 회색
- 질문형 문구 사용 권장

### 보조 인프라 / 외부 연결
- 회색

---

## 3. 텍스트 규칙

- 줄 수는 2~3줄 이내 권장
- 긴 문장은 본문으로 보내고, 다이어그램에는 역할만 남긴다
- 한 노드 안에 한국어/영어를 섞을 때는
  - 첫 줄: 핵심 역할
  - 둘째 줄: 보충 설명
패턴을 유지한다

예:
- `Agent stream<br/>~300ms first token`
- `StateBackend<br/>(기본값)`

---

## 4. 구조 규칙

### 초광폭 다이어그램
- 가능하면 `LR`보다 `TD` 또는 2단 구조를 우선 고려
- 한 페이지에 넣기 위해 지나치게 축소되는 구조는 피한다

### 세로형 다이어그램
- 단계 수가 많아도 6~8단 정도를 넘기면 묶음 또는 subgraph로 요약

### 비교 다이어그램
- 같은 레벨끼리 가로 정렬
- 선택지는 세로 스택보다 병렬 비교가 읽기 쉽다

---

## 5. 현재 Book 적용 원칙 요약

- 민트 = 기본 개념 / 결과 / 정상 경로
- 코랄 = 핵심 실행 / 활성 단계
- 노랑 = 판단 / 조정 / 승인 / 제어
- 회색 = 보조 정보 / 외부 요소 / note

---

## 6. 운영 원칙

새 Mermaid를 추가할 때는 다음 순서로 확인한다:

1. 이 노드는 **개념/결과**인가? → 민트
2. **실행 중심 단계**인가? → 코랄
3. **판단/제어 지점**인가? → 노랑
4. **설명/보조 요소**인가? → 회색

이 규칙으로 먼저 배정하고, 예외는 최소화한다.
